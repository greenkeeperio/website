01 How it works

Greenkeeper sits between npm and GitHub, observing all of the modules you depend on. When they get updated, your project gets a pull request with that update. Your CI tests kick in, and Greenkeeper watches them to see whether they pass. If they do, you can just click the merge button, and we'll even auto-close branches for older versions for you. If not, you can use the branch to fix things.

You‚Äôll be amazed at how many dependency updates you‚Äôve been missing.

Let the friendly Greenkeeper robot take all the dull work of keeping your dependencies up to date off your shoulders and, optimally, boil it all down to a single click. This is as close to fully automatic as we could possibly make it.

Why it‚Äôs so great

  • Staying on top of dependency updates is a lot of cognitive overhead: your code depends on modules that depend on modules. It's modules all the way down. Stuff gets updated all the time. It‚Äôll either annoy you, or you‚Äôll miss it.
  • Version pinning would save you the hassle, but you‚Äôll miss bugfixes and new features, and when you do have to update everything in the future, it‚Äôll probably be a huge mess. So let‚Äôs not pin.
  • Version ranges would be nice, but suffer from the fact that SemVer has‚Ä¶ room for interpretation, and module authors simply make mistakes from time to time.
  • You could do all the updates by hand, all the time. But that‚Äôs dull, repetetive work, which is why, let‚Äôs be honest, barely anyone actually does it.
